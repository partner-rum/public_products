# План работ: витрина структурных продуктов

Цель: публичная витрина, которую сейлзы пополняют сами — без разработчика в цепочке.

## Архитектура (принцип)

Статический сайт остаётся статическим. Единственное, что меняется день ото дня, —
данные (`data/instruments.js`). Значит, задача «сейлзы вносят сами» решается не CMS-кой,
а понятным им источником данных (таблицей) и автоматикой, которая превращает таблицу в файл данных.

```
Google Sheets (сейлзы)  →  GitHub Actions (по расписанию/кнопке)  →  update_site.py
                                                                        ↓
                                                              data/instruments.js
                                                                        ↓
                                                              GitHub Pages (сайт)
```

## Этап 1 — Публикация (0,5–1 день)

1. Создать репозиторий на GitHub, залить содержимое папки.
2. Включить GitHub Pages: Settings → Pages → Deploy from a branch → main / root.
3. Домен: купить/выделить поддомен, например `products.example.com`,
   CNAME на `<логин>.github.io`, вписать в Settings → Pages → Custom domain (+ Enforce HTTPS).
4. Favicon (золотая звезда из шапки), Яндекс.Метрика.

**Важно:** GitHub Pages — только публичные сайты. Если витрина должна быть закрытой
(только по ссылке от сейлза — нормально; под паролем — нет), нужен другой хостинг:
Cloudflare Pages + Access, либо VPS/российский хостинг с basic auth.

## Этап 2 — Самообслуживание сейлзов (1–2 дня)

1. **Google-таблица** — единый прайс-лист. Колонки = поля инструмента:
   `id, type, structure, name, underlying, cls, uRef, strike, strike2, tenor, expiry, quote, chg, minNom, показывать(да/нет)`.
   Валидация прямо в таблице: выпадающие списки для `type`/`structure`, формат дат, защита колонок.
   Отдельный лист — базовые активы (`underlying, unit, desc, history`).
2. **Доработать `update_site.py`**: читать CSV-экспорт таблицы
   (`https://docs.google.com/spreadsheets/d/<ID>/export?format=csv&gid=<GID>`),
   валидировать (обязательные поля, страйки, даты — падать с понятной ошибкой),
   генерировать `data/instruments.js` как сейчас.
3. **GitHub Action**: запуск по расписанию (например, каждый час в рабочее время)
   и вручную кнопкой «Run workflow». Скрипт → коммит `instruments.js` → Pages пересобирается.
4. **Инструкция для сейлзов** (полстраницы): «правишь таблицу → через N минут на сайте».

Итог: сейлз добавляет строку в привычную таблицу — инструмент появляется на доске,
карточка собирается автоматически (тексты, диаграмма, пример расчёта — по типу продукта).

Запасной вариант без Google: `data/instruments.csv` в самом репозитории,
сейлзы правят через веб-интерфейс GitHub, Action пересобирает по пушу.

## Этап 3 — Рост (по мере надобности)

- **Новые типы продуктов** (автоколлы, ноты с купоном и т.д.): добавить тип в `data/lib.js`
  (тексты, формула) и ветку в `instrument.html` (диаграмма, сценарии). Данные — те же колонки таблицы.
- **SEO / «весь рынок»**: когда инструментов станут десятки — генерировать отдельную
  HTML-страницу на инструмент (расширить `update_site.py` или перейти на Astro).
  Каждый инструмент получит свой URL для поиска и для ссылок сейлзов.
- **Заявка с карточки**: кнопка «Обсудить с менеджером» → форма → уведомление
  в Telegram-чат сейлзов (бот + Action/сервис). Витрина начинает приносить лиды.
- **Термшит в PDF**: печатная версия карточки уже свёрстана (Ctrl+P на странице инструмента);
  можно добавить кнопку «Скачать PDF».
- **Комплаенс**: дисклеймеры, правовая информация, пометка «не оферта» — согласовать с юристами
  до публичного запуска.
- **Реальные котировки/история**: когда появится источник (прайсер), `chg` и историю
  писать из него же через тот же пайплайн.

## Что уже сделано

- Дизайн-система «полярная ночь»: Literata + Onest + JetBrains Mono, тёмный переплёт,
  фамильные цвета семейств, линия выплаты как фирменный мотив.
- Три страницы: главная (морфинг профилей выплат), доска (матрица варрантов,
  переключатель семейств), карточка (диаграмма выплат, пример расчёта, печатная версия).
- Данные отделены от вёрстки: `data/instruments.js` (генерируется) + `data/lib.js` (логика).
